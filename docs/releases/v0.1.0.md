# GoClaude v0.1.0

> üöÄ First public release ‚Äî June 9, 2026

GoClaude is a **terminal AI coding assistant**, rewritten in Go from the upstream
TypeScript claude-code. It follows strict **DDD four-layer architecture** with
dependency inversion, targeting modularity, testability, and zero-framework bloat.

```bash
make all && ./bin/goclaude
```

### Tools

14+ built-in tools registered with the domain tool registry:

| Category | Tools |
|----------|-------|
| Filesystem | `file_read`, `file_write`, `file_edit`, `glob`, `grep` |
| Execution | `bash` (sandboxed) |
| AI | `agent` (subagent), `skill`, `team` |
| External | `mcp` (any MCP server), `web_search`, `web_fetch` |
| Utility | `todo_write`, `ask_user` |

Read-only tools run concurrently (max 10), write tools serialize with
permission checks.

### MCP Protocol

Connect external tools via the Model Context Protocol. Four transports:

| Transport | Use case |
|-----------|----------|
| `stdio` | Local MCP servers (npx, uvx) |
| `HTTP` | REST-style MCP endpoints |
| `SSE` | Server-sent event streaming |
| `WebSocket` | Persistent bi-directional |

Tools appear as `mcp__postgres__query`, `mcp__github__create_issue`, etc.
Configure in `.goclaude/.mcp.json` or user `~/.goclaude/settings.json`.

### Multi-Agent Teams

Enable team mode with `--agent-teams`. The coordinator agent decomposes complex
work, assigns tasks to workers, and merges results automatically.

```
> Build a REST API skeleton: auth, user CRUD, middleware

‚öô team_create: backend-api
‚öô create_task ‚Üí worker-1: auth module
‚öô create_task ‚Üí worker-2: user CRUD
‚öô create_task ‚Üí worker-3: middleware

[worker-1] ‚úì auth complete
[worker-2] ‚úì CRUD complete
[worker-3] ‚úì middleware complete
```

Natural language routing eliminates manual task dispatch. Manage teams via the
`/teams` REPL command or `goclaude team` CLI.

### Skills

Encapsulate expert prompts as reusable, conditionally-activated skill packages.

```markdown
---
name: "API Reviewer"
description: "Review REST APIs for consistency and security"
when_to_use: "When reviewing HTTP endpoints"
paths: ["**/handler/**", "**/middleware/**"]
allowed-tools: ["file_read", "grep", "bash"]
---

You are an API design reviewer. Check for RESTful conventions,
input validation, auth middleware, and consistent error handling.
```

Skills load from project `.goclaude/skills/`, user `~/.goclaude/skills/`, and
legacy `.claude/skills/`. The `paths` field triggers automatic activation when
the AI reads matching files.

### Rules

Persistent behavior constraints auto-injected into every session:

```markdown
# ~/.goclaude/rules/preferences.md

## Language
- Always respond in Chinese
- Code comments in English
```

Rules stack: user-level rules load first, project-level rules append on top ‚Äî
no overwrites, pure accumulation.

### Memory

Cross-session memory persists what the AI learns about your project. Stored in
`~/.goclaude/projects/<sanitized-path>/memory/` ‚Äî never in your git repo.

- AI autonomously calls `update_memory` to save context
- `/remember` reviews, deduplicates, and promotes entries to CLAUDE.md
- Enforce size limits via `max_memory_lines` and `max_memory_bytes` config

### Hook System

Intercept 7 lifecycle events with YAML-declared hooks:

```yaml
hooks:
  hooks_config:
    matchers:
      - event: PreToolUse
        tool_name: bash
        hook_name: pre-bash-audit
        command: "python3 ~/.goclaude/hooks/bash_audit.py"
      - event: SessionStart
        hook_name: session-init
        command: "cat ~/.goclaude/context_guide.md"
```

Hooks receive JSON context via stdin, return JSON results via stdout. Multiple
handlers run sequentially per event; any handler can block execution by
returning `{ "block": true }`.

### Permission System

| Mode | Behavior |
|------|----------|
| `default` | Ask for each write operation |
| `acceptEdits` | Auto-approve file edits, ask for bash |
| `plan` | Plan-only mode, no writes |
| `bypass` | Skip all permission checks |

Shift+Tab toggles modes live in the REPL. Path-based rules allow fine-grained
control (e.g., auto-approve all writes under `./pkg/`).

### Sandbox

Linux uses [bubblewrap](https://github.com/containers/bubblewrap) (`bwrap`),
macOS uses native `sandbox-exec`. Configurable:

```yaml
sandbox:
  enabled: true
  filesystem_write:
    allow: ["./", "~/.goclaude/tmp"]
    deny: ["~/.ssh", "~/.aws"]
  network:
    disable_network: false
```

### Architecture

```text
interfaces ‚Üí application ‚Üí domain ‚Üê infrastructure
```

| Layer | Responsibility | Dependencies |
|-------|---------------|-------------|
| **Domain** | Pure interfaces, models, business logic | Zero external deps |
| **Application** | Orchestrates domain services into use cases | Domain only |
| **Infrastructure** | Implements domain interfaces (API, FS, MCP, Git) | Domain + I/O |
| **Interfaces** | CLI (Cobra), REPL (Bubbletea), TUI | Application |
